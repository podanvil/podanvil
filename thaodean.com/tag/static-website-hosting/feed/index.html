In this blog post, we will explore how to host a simple static website using Amazon S3 and set up a custom domain using Amazon Route 53.

Step 1: Create an Amazon S3 Bucket

First, we need to create an S3 bucket where our website’s files will be stored. Log into your AWS Console and navigate to the S3 service. Click on “Create bucket” and provide a unique name for your bucket. Remember to uncheck “Block all public access” if you want your website to be publicly accessible.

Step 2: Upload Your Website Files

Next, upload your website’s files to the S3 bucket. You can do this by clicking on the name of your bucket to open it, then clicking “Upload”. Select the files you want to upload (for a basic website, this might be an index.html file) and click “Upload”.

Step 3: Set Bucket Policy

To allow public access to the files in your bucket, you need to set a bucket policy that permits this. In the S3 bucket settings, navigate to “Permissions” and then “Bucket Policy”. Here, you can set a policy that allows public read access to your files.

Here is a sample policy that does this:

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::{your-bucket-name}/*"
        }
    ]
}

Replace “{your-bucket-name}” with the actual name of your S3 bucket.

Step 4: Register a Domain with Amazon Route 53

Now, we need a domain that users can use to access your website. Amazon Route 53 is a domain registration service that integrates well with S3.

In the AWS Console, navigate to the Route 53 service and click on “Register Domain”. Follow the steps to register a domain. Once your domain is successfully registered, you can configure it to point to your S3 bucket.

Step 5: Configure DNS Settings

In Route 53, navigate to “Hosted zones” and click on the name of your domain. Create a new record set, select the type as “A – IPv4 address”, and select “Alias”. In the alias target, enter the S3 website endpoint that AWS provided when you created your bucket. Save the record set.

Now, your domain should be pointing to your S3 bucket. When you navigate to your domain in a web browser, you should see your website!

Conclusion

Amazon S3 and Route 53 are powerful tools that allow you to host and serve static websites. By leveraging these services, you can have your website up and running in no time!
